## 🎯 Что это

ClickJurist — это клиент-серверное приложение, которое на заданный юридический вопрос пользователя
выдаёт структурированную консультацию на русском языке со ссылками на нормы законодательства РФ,
а по запросу — чек-лист действий и шаблон документа (иск / претензия / жалоба) с возможностью скачивания в PDF.

Главная техническая находка — **двухэтапный LLM-пайплайн**:

```
Юрист-консультант (gemini-2.5-flash)  →  Senior-асессор (minimax-m3)  →  Пользователь
   быстрый черновик                      проверка + финальная сборка      только «чистый» эталон
```

LLM-1 пишет быстрый черновик, LLM-2 проверяет и переписывает, отбрасывая галлюцинации и «воду».

## 🚀 Запуск локально

### 1. Установить зависимости

```bash
cd clerk_craft
python -m venv .venv
.venv\Scripts\activate
pip install -r backend/requirements.txt
```

### 2. Настроить переменные окружения

Создайте файл `clerk_craft/.env` на основе `.env.example` (или просто заполните свои):

```env
AITUNNEL_API_KEY=sk-ваш-ключ
LLM_DRAFT_MODEL=gemini-2.5-flash
LLM_ASSESSOR_MODEL=minimax-m3
AITUNNEL_BASE_URL=https://api.aitunnel.ru/v1
```

### 3. Запустить сервер

```bash
cd clerk_craft
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

Откройте в браузере: **http://localhost:8000**

## 🧠 LLM-пайплайн (подробности)

### Этап 1 · Юрист-консультант (`gemini-2.5-flash`)

Системный промпт задаёт роль квалифицированного юриста РФ с жёсткими правилами:
- «не придумывай» номера дел, суммы, пошлины;
- «не выдумывай» статьи — пиши плейсхолдеры;
- объём ≤ 150 слов;
- только кириллица;
- анонимность персональных данных;
- если данных не хватает — задай 1–2 уточняющих вопроса.

### Этап 2 · Senior-асессор (`minimax-m3`)

Асессор делает **5 внутренних проверок** черновика LLM-1:

1. Нормативная база — есть ли актуальные ссылки на ГК/УК/ТК/КоАП РФ.
2. Судебная практика — упоминание позиции ВС РФ.
3. Расчёты и сроки — корректны ли (госпошлины, исковая давность, сроки уведомлений).
4. Анти-вода — убрать дисклеймеры, повторы, вводные фразы.
5. Сборка финального ответа 400–700 слов.

**Жёсткое правило:** в ответе модель показывает **только шаг 5**. Шаги 1–4 остаются «в голове».

### Страховка в коде: `_strip_to_final_answer()`

Даже если LLM-2 просочит служебные шаги 1–4, функция в `llm_chain.py` оставляет только блок
после `## Итоговый ответ` — пользователь получает чистый результат.

## 🛡 Безопасность и приватность

- `.env` с API-ключом **исключён** из репозитория через `.gitignore`.
- Системные промпты требуют от LLM не выдумывать номера статей и ФИО, использовать плейсхолдеры.
- Шаблоны документов также используют плейсхолдеры `<ФИО>`, `<АДРЕС>`, `<ДАТА>` — пользователь
  заполняет их сам перед подачей.
